# Internationalisation

HEXA Structures utilise l'internationalisation Qt côté GUI uniquement :

- `QTranslator` charge les fichiers compilés `.qm`.
- Les sources de traduction sont les fichiers `.ts` dans `i18n/`.
- Le français reste la langue source par défaut.
- Le core métier ne dépend pas de PySide6 pour les traductions.

Pour l'instant, seules les langues prêtes sont visibles dans le menu de l'application (`fr`, `en`). Les fichiers `hexa_es.*` et `hexa_de.*` restent présents comme squelettes mais ne sont pas exposés tant que les traductions ne sont pas suffisamment complètes.

## Ajouter une langue

1. Ajouter la langue dans `LanguageManager.available_languages` :

```python
available_languages = {
    "fr": "Français",
    "en": "English",
    "es": "Español",
    "de": "Deutsch",
    "it": "Italiano",
}
```

2. Générer le fichier `.ts` depuis le code GUI.

Sur ce projet, passer explicitement les fichiers `.py` et `.ui` à `lupdate` pour ramasser les widgets Python et les formulaires Qt Designer :

```powershell
$files = @(Get-ChildItem -Path gui -Recurse -Include *.py,*.ui | ForEach-Object { $_.FullName })
pyside6-lupdate @files -ts i18n/hexa_en.ts
```

Pour une autre langue :

```powershell
$files = @(Get-ChildItem -Path gui -Recurse -Include *.py,*.ui | ForEach-Object { $_.FullName })
pyside6-lupdate @files -ts i18n/hexa_es.ts
```

La forme courte reste utile pour certains environnements, mais elle peut ne pas scanner récursivement tous les fichiers Python selon la version de PySide :

```powershell
pyside6-lupdate gui -ts i18n/hexa_en.ts
```

3. Ouvrir le fichier `.ts` dans Qt Linguist ou un éditeur compatible, puis renseigner les traductions.

4. Compiler le fichier `.qm` :

```powershell
pyside6-lrelease i18n/hexa_en.ts -qm i18n/hexa_en.qm
```

Pour une autre langue :

```powershell
pyside6-lrelease i18n/hexa_es.ts -qm i18n/hexa_es.qm
```

## Convention de nommage

Les fichiers suivent la convention :

```text
i18n/hexa_fr.ts
i18n/hexa_en.ts
i18n/hexa_es.ts
i18n/hexa_de.ts

i18n/hexa_fr.qm
i18n/hexa_en.qm
i18n/hexa_es.qm
i18n/hexa_de.qm
```

Le chargeur utilise le code langue pour construire le chemin :

```python
qm_path = i18n_dir / f"hexa_{language_code}.qm"
```

## Packaging

Le spec PyInstaller inclut déjà le dossier :

```python
("i18n", "i18n")
```

Pour un build Nuitka, ajouter l'option équivalente :

```powershell
--include-data-dir=i18n=i18n
```

## Bonnes pratiques

- Utiliser `self.tr("Texte visible")` dans les widgets PySide6.
- Ne pas traduire le core métier directement.
- Préférer des codes métier côté core, puis traduire dans la GUI plus tard, par exemple `plate_requires_four_nodes`, `section_missing`, `invalid_material`.
- Ajouter les textes progressivement : menus, actions, panneaux principaux, puis dialogues et messages.
